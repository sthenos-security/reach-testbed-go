CI Evidence Flow

The workflow is branch-first. It creates evidence before remediation, runs the agent only when enabled, then creates separate proof evidence from the remediation branch.

Step Workflow/script Why it exists
Install and cachereachable-remediate.yml, ci/reachable-cache-evidence.pyInstalls or upgrades Reachable, records whether the run reused cache or started fresh, and captures version/database evidence.
Baseline scanreachctl scan --ci, ci/collect-reachable-report.shScans the vulnerable branch and writes the baseline repo.db evidence used by the demo contract.
Baseline contract checkexpected/baseline.json, ci/check-db-remediation-proof.pyVerifies that the known vulnerable branch still matches the expected findings before any remediation is trusted.
Remediation taskreachctl remediate, ci/run-agent.shSynthesizes a bounded task from DB evidence and asks the selected agent to patch a dedicated remediation branch.
Project safety gatego test ./... by defaultConfirms the agent did not break the application before Reachable performs the proof scan.
Proof scanreachctl scan --ci on the remediation branchCreates a second DB-backed scan from the patched branch so fixes are measured from evidence, not from the agent's claims.
DB verdictci/check-db-remediation-proof.py, db-remediation-verdict.jsonCompares baseline and proof databases. The run passes only when expected findings are fixed and remaining actionable rows are zero.
Publish evidenceci/build-pages-summary.py, GitHub PagesPublishes a sanitized report with branch, commit, scan IDs, runtime, cache state, AI cost, and artifact links. Private prompts and raw databases are not published.

CI Controls

These controls are implemented by the Reachable Remediation Template workflow. They define whether CI only scans, creates a remediation branch, opens a pull request, or verifies an existing branch.

Control Default Why it matters
remediatefalseMain kill switch. The default workflow scans and proves without changing code.
remediation_modecodex-openaiOne-key CI mode. Use OPENAI_API_KEY for Codex or ANTHROPIC_API_KEY for Claude.
max_batches1Prevents sending an unbounded backlog to one prompt.
proof_rescanEnabledReachable performs a separate proof scan of the remediation branch. The final database must contain no production-actionable results.
rescan_onlyfalseWhen true, verifies an existing remediation branch without invoking a coding agent or changing code.
public_artifactsDB verdict + sanitized ledgerRaw prompt bundles, rules, skills, fuzz/pentest prompts, agent transcripts, and local databases are not uploaded.
